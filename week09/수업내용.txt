셸 : 명령어 해석기 : 우리가 친 명령어를 해석해서 컴퓨터를 작동시킨다.
	자체 프로그래밍 기능이 있어서 코드를 통해서 자동화를 시킬 수 있다.

본 쉘(sh) :  유닉스의 기본 사용 쉘
콘 쉘 : AT&T사에서 만들었고 본셸을 확장해서 만들었다.
배쉬셸(bash) : 리눅스 표준 셸, ksh와 csh의 장점을 결합 > 패스워드 파일에서 일반, 루트 유저들이 배쉬셸을 쓰고 있는 걸 알 수있다.

지 셸 : 콘 셸을 재작성해서 업그레이드

C셸 : C언어의 특징을 많이 포함해서 c 셸이다. 

chsh : change shell : 유저 본인이 자신의 셸을 바꾸고 싶을 때 사용한다.  
usermod : 관리자의 입장에서 사용자들의 셸을 전부 바꿔준다. 
-s 지정하는 셸을 앞으로 로그인할 셸로 변경한다.
-l 내가 사용할 수 있는 쉘의 목록을 출력해줘.

환경 변수(전역 변수)
- 전체 셸에서 사용 가능한 전역 변수
env : 모든 환경 변수 확인 명령 
② 셀 변수(지역 변수)
- 현재 로그인 셸에서만 사용 가능한 지역 변수
set :  셸 변수 개별 확인 명령

셸 내의 변수가 지역, 전역변수로 나눌 수 있고, 리눅스의 기본 셸인 배쉬셸의 
시작은 아래의 정리본에 있다.

/etc/profile : 모든 사용자의 셸 환경을 제어하는 전역적인 시스템 설정 파일
/etc/bashrc : bash 수행 시 실행되는 함수를 전역적으로 제어
~/.bash_profile : 개별 사용자의 셸 환경을 설정하는 파일로 로그인 시 읽어 들임
~/.bashrc : bash 수행 시 실행되는 함수를 지역적으로 제어
~/.bash_logout : 로그아웃하기 직전에 실행되는 시스템 설정 파일
~/.bash_history : 키보드로 입력한 명령어 내용들을 파일 안에 기록

셸 파일은 전역, 지역적 파일로 나뉘어 지고, ls -a를 하면 숨겨진 지역 파일까지 나온다.
④ /etc/profile.d에서 ".d"는 디렉토라는 의미입니다.
